# 存储功能说明

## 实现文件

主要文件：

```text
mercury_storage.py
migrations/
```

项目使用 SQLite 保存订阅、文章、清洗内容、标签和配置。数据库默认位置：

```text
~/.mercury_pyqt/mercury.db
```

## 已实现表结构

核心表：

1. `feeds`
   - 保存订阅源。
   - 字段包括 `feed_id`、`title`、`url`、`added_at`。
2. `entries`
   - 保存文章条目。
   - 字段包括 `entry_id`、`feed_id`、`stable_id`、`title`、`url`、`summary`、`is_starred`、`is_unread`。
3. `contents`
   - 保存清洗结果。
   - 字段包括 `source_html`、`cleaned_html`、`canonical_markdown`、`reader_html`。
4. `tags`
   - 保存文章标签。
   - 支持一个文章多个标签。
5. `settings`
   - 保存轻量配置。
6. `summary_results`、`translation_segments`、`provider_profiles`、`agent_runs`
   - 已有迁移表结构，作为后续 AI 功能预留。

## 已实现 Store

1. `FeedStore`
   - 新增 / 更新 Feed。
   - 查询 Feed。
   - 删除 Feed。
2. `EntryStore`
   - 新增 / 更新文章。
   - 按 Feed、未读、收藏筛选文章。
   - 标记已读 / 未读。
   - 收藏 / 取消收藏。
   - 标签增删和标签筛选。
3. `ContentStore`
   - 保存清洗后的 `ReaderDocument`。
   - 查询清洗缓存。
4. `SettingsStore`
   - 保存轻量配置。
5. `StorageService`
   - 面向 GUI 的统一服务接口。

## 已实现功能

1. 本地优先：
   - 用户数据保存在本机 SQLite。
   - 不依赖云端数据库。
2. Feed 持久化：
   - 添加订阅后保存到 `feeds`。
   - 刷新后文章保存到 `entries`。
3. 文章状态：
   - 收藏状态保存到 `is_starred`。
   - 已读状态保存到 `is_unread`。
4. 标签：
   - 标签保存到 `tags`。
   - 支持标签列表、标签筛选、添加标签、移除标签。
5. 清洗缓存：
   - 清洗后的 HTML / Markdown / Reader HTML 保存到 `contents`。
   - 再次打开文章优先使用缓存。
6. 删除订阅：
   - 删除 Feed 时级联删除文章、清洗缓存和标签。
   - 支持批量删除订阅。
7. 后台线程安全：
   - `create_worker_copy()` 会创建同数据库路径的新连接。
   - 后台刷新和批量清洗不复用 GUI 线程的 SQLite connection。

## 迁移

迁移文件位于：

```text
migrations/
```

启动 `StorageService` 时会自动调用 yoyo migrations，确保数据库结构更新到当前版本。

## 已知限制

1. 当前使用 sqlite3 标准库，不是 SQLAlchemy。
2. 暂未实现数据库压缩和清理策略。
3. 暂未实现清洗缓存单独删除。
4. API Key 存储尚未实现，后续应使用系统 keyring。
5. 打包后首次启动如迁移失败，需要检查 DMG 中是否包含 `migrations/`。
