# 文章清洗功能说明

## 实现文件

主要文件：

```text
reader/fetcher.py
reader/readability.py
reader/sanitizer.py
reader/markdown_converter.py
reader/html_renderer.py
reader/pipeline.py
```

清洗功能由 `ReaderPipelineService` 统一调度。

## 已实现功能

1. 抓取源 HTML：
   - 使用 `httpx` 请求文章 URL。
   - 支持重定向。
   - 记录原始 URL 和最终 URL。
2. 正文提取：
   - 使用 `readability-lxml` 提取正文。
   - 获取 readable title 和正文 HTML。
3. HTML 清洗：
   - 使用 BeautifulSoup 修复链接和图片地址。
   - 使用 bleach 移除脚本、危险属性和不安全协议。
   - 支持懒加载图片属性：
     - `data-src`
     - `data-original`
     - `data-lazy-src`
     - `data-url`
     - `srcset`
     - `data-srcset`
4. Markdown 转换：
   - 使用 `markdownify` 将 cleaned HTML 转换为 canonical Markdown。
   - 规范空行和行尾空白。
5. Reader HTML 渲染：
   - 使用 `markdown-it-py` 将 Markdown 渲染为 HTML。
   - 清洗后使用精致 Reader 样式。
   - 未清洗/还原时使用朴素样式。
6. 图片展示：
   - Reader 使用自定义 `ReaderTextBrowser`。
   - 对远程 http/https 图片进行显式加载和缓存。

## 清洗产物

清洗后会生成 `ReaderDocument`：

```text
source_html
cleaned_html
canonical_markdown
reader_html
```

这些内容会保存到 SQLite 的 `contents` 表。

## 主要流程

```text
article.url
  -> SourceHtmlFetcher.fetch()
  -> extract_readable_html()
  -> clean_reader_html()
  -> html_to_markdown()
  -> render_markdown_to_reader_html()
  -> ContentStore.save()
```

## GUI 行为

1. 点击“清洗”后后台线程执行清洗。
2. 清洗成功不弹窗，只刷新 Reader 区和底部状态栏。
3. 清洗失败时弹出错误窗口。
4. 再次打开已清洗文章时，优先读取缓存的 `reader_html`。
5. 点击“还原”会临时显示未清洗前的朴素样式。

## 已知限制

1. 动态 JS 页面可能无法抓取完整正文。
2. 某些网站会反爬或需要登录，清洗可能失败。
3. 图片防盗链可能导致图片无法显示。
4. 批量清洗耗时较长，目前逐篇处理。
5. 当前没有清洗缓存删除入口。
