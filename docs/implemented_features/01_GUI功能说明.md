# GUI 功能说明

## 实现文件

主要文件：

```text
mercury_gui.py
```

GUI 使用 PySide6 实现，主窗口类为 `MercuryMainWindow`，应用显示名称为 `Lumen`。

## 已实现功能

1. 三栏布局：
   - 左侧：Feeds / Tags 侧边栏。
   - 中间：文章列表。
   - 右侧：Reader 阅读区。
2. 工具栏功能：
   - 刷新订阅。
   - 导入 OPML。
   - 删除订阅。
   - 批量删除订阅。
   - 清洗文章。
   - 还原显示。
   - 收藏 / 取消收藏。
   - 标记已读 / 未读。
   - 添加 / 移除标签。
   - 标签批量标已读、批量收藏、批量清洗。
3. 订阅侧边栏：
   - 显示 All Feeds、Starred 和真实订阅。
   - 显示未读数量。
   - 默认隐藏批量选择框，进入批量选择模式后才显示。
   - 支持勾选多个订阅后批量删除或取消批量选择。
4. 标签侧边栏：
   - 显示所有标签和文章数量。
   - 点击标签后筛选文章列表。
5. 文章列表：
   - 显示标题、Feed 名、发布时间、已读/未读状态。
   - 未读文章标题加粗。
   - 点击文章自动标记为已读。
   - 点击星星可收藏 / 取消收藏。
6. Reader 区：
   - 未清洗前显示文章摘要/原始 Markdown。
   - 清洗后显示 Reader 样式 HTML。
   - 支持远程图片加载。
   - 点击还原可恢复未清洗显示。
7. 后台任务：
   - 刷新订阅在后台线程执行。
   - 清洗文章在后台线程执行。
   - 标签批量清洗在后台线程执行。

## 主要交互流程

### 阅读文章

1. 用户选择左侧订阅。
2. 中间文章列表刷新。
3. 用户点击文章。
4. Reader 区显示文章。
5. 如果文章是未读，则自动写入已读状态。

### 清洗文章

1. 用户选择文章。
2. 点击“清洗”。
3. 后台线程抓取网页并清洗正文。
4. 清洗结果保存到 SQLite。
5. Reader 区显示清洗后的 HTML。

### 批量删除订阅

1. 点击工具栏中的“批量选择”。
2. 在 Feeds 侧边栏勾选多个真实订阅。
3. 点击“删除已选”并确认。
4. 从数据库删除订阅，文章、清洗缓存和标签会随外键级联删除。
5. 删除完成后自动退出批量选择模式，也可点击“取消批量”退出。

## 已知限制

1. UI 仍是单文件主窗口实现，后续可拆分为 `feed_sidebar.py`、`entry_list.py`、`reader_view.py`。
2. 摘要和翻译按钮仍是占位逻辑。
3. 批量清洗会逐篇抓取网页，耗时取决于网络和网站响应。
4. 当前没有系统托盘和全局快捷键。
